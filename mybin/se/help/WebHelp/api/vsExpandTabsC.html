void VSAPI vsExpandTabsC(int wid, char *pszDest, int *pDestLen, int StartCol, int ColWidth, char Option)

Takes sub-string of source buffer and places the result in the dest buffer. Unlike the the vsExpandTabs function, this function gets the source text from the current line.

The StartCol and ColWidth specification correspond to the input string as if tab characters were expanded according to the buffers tab settings. We call this type of text position or count imaginary. Strings containing tab characters are expanded before displayed. Hence, the need arises for a differentiation between physical and imaginary positions. A physical position corresponds to a byte in a string where the characters are numbered one to the length of string. An imaginary position corresponds to a position in a string once tabs have been expanded.

Parameters:

wid - wid identifies the object to be operated on. 0 specifies the current object.

pszDest - Output. Resulting string terminated with ASCII 0.

pDestLen - Output. Number of bytes written to pszDest not including ASCII 0.

StartCol - Input. Start (imaginary) column for sub- string.

ColWidth - Input. Number of (imaginary) columns to get from pSource. -1 specifies the rest or the string. -2 specifies to use the vsTruncQLineLength instead of the real line length. This has the same effect as specifying -1 if the VSP_TRUNCATELENGTH property is 0.

Option - Input. Specify 'E' if you want all tabs expanded. Otherwise specify 'S' to return tabs unexpanded.

Applies To:

Edit Window, Editor Control

Categories:

Edit Window Methods, Editor Control Methods, String Functions

See Also:

vsExpandTabs, vsTextColC
