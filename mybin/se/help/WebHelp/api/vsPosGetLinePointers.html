void VSAPI vsPosGetLinePointers(unsigned char **pp, unsigned char **ppBeginLine, unsigned char **ppEndLine, seSeekPos *pRelLine)

You must call vsPosInit before calling this function. Once you use a non vsPosXXX function which accesses buffer data you MUST call vsPosInit again to avoid crashing the editor. Only use this function if you want your searching to be as fast as possible.

This function is used to quickly scan part of the current Pos line (Not the same as the cursor position).

Parameters:

pp - Set to point to current byte. You must check that pp is within the range *ppBeginLine and *ppEndLine.

ppBeginLine - Set to first valid byte at the beginning of the current Pos line segment.

ppEndLine - Set to point past the last valid byte of the current Pos line segment.

pRelLine - Use this to track the relative number of lines the Pos cursor as moved up or down. Subtract the initial RelLine from the RelLine when you are done to get the number of complete lines the Pos cursor has gone up or down.

Applies To:

Edit Window, Editor Control

Category:

Search Functions
