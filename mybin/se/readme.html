Fixes, Features, and Enhancements in SlickEdit 2013 (18.0.1)

In addition to the items below, all hotfixes released for v18.0.0 have been incorporated into this release
  • Platform Support
    • SlickEdit 2013 is now available for AIX, Solaris SPARC, and HP-UX
  • New MDI Architecture
    • Added the ability to drop files into a specific tab group when dragging from the filesystem browser (Windows Explorer, Mac Finder, etc)
    • Fixes for Unix platforms that had excessive flicker when switching focus to/from a floating document tab window
    • Corrected behavior of abbreviated document tabs when the tab order is changed
    • Corrected behavior of the next-tab-group and prev-tab-group commands so that the active document tab is preserved
    • Improved appearance of the drag-drop guide overlay
  • Version Control
    • Usability enhancements for the new History Diff tool window, including display of revision comments and sizeable panes
    • Added an option to select changelist for Perforce operations
    • Fix for Git when source controlled files are not in the current working directory
    • Corrected cases where Perforce diff would inject an extra line
    • Fixed behavior of mouse wheel scrolling in the version control diff dialog
    • Several fixes for CVS, including cvs status reporting and handling of CVSROOT
  • Language Support
    • Added initial support for Scala
    • Improvements to tagging and auto-completion for Objective-C
    • Added Context Tagging for the TTCN-3 language
    • Fix for cases where navigating to references with find-next could cause a crash
  • Beautifier Enhancements
    • Corrected hotspot placement in C# syntax expansions and alias expansions
    • Better support for comment wrapping Doxygen-style comments
    • Fixed cases where preprocessor header guards with leading comments were not properly detected
    • Fixed bug in the "Force if complex" option for adding/removing return statement parens
  • General
    • Fixed a Print Preview crash that affected some Unix platforms
    • Corrected issue with special character display when lines are scrolled horizontally
    • Automated XML schema mapping for MSBUILD files
    • Prevent corruption of alias files when inserting new aliases
    • Better detection of build message type (errors vs warnings) when inserting build output into the Message List tool window
    • Improved handling of document tabs context menu when using Cygwin X-Windows client

Original Release Notes for SlickEdit 2013 (18.0.0)

  • New MDI Architecture
    • The MDI layout has been completely rewritten to support multiple groups of document tabs
    • Document tabs (and groups of tabs) can be floated as separate top-level windows, allowing better use of multiple monitors
    • Files can be dragged and dropped to move between tab groups or to create new groups
    • Resizing the outer MDI window automatically resizes the document tab edit windows. This ensures you utilize all the available space in the MDI area.
    • The new document tabs offer much better performance than the previous File Tabs tool window, better supporting users who prefer to keep many files open
  • Version Control
    • A new Repository Browser dialog displays a summary of a repository's comment log history. Use the Tools > Version Control > Browse Repository... menu to bring up the dialog
    • A new History Diff dialog allows you to quickly see side-by-side historical revision differences. This is supported for Subversion, Perforce, Mercurial, Git, and Backup History. Use the Tools > Version Control > History Diff on... menu.
  • Language Support
    • Support for Matlab, CoffeeScript, Markdown, Google Go, and TTCN-3 has been added
    • Tagging and source navigation of CSS is improved
    • Color coding support for the Cg shader language has been added
  • Beautifier Enhancements
    • Java and C# are now supported in the new beautifier framework.
    • New options have been added to support beautification for JavaDoc, Doxygen, and XMLDoc comments.
  • Project Support and Debugging
    • Updated builds of GDB 7.5 (with Python pretty-printing support) are now distributed under the SlickEdit installation directory
    • Variables now appear in declarative order in the Debug Autos, Locals, and Members tool windows
    • Setting breakpoints in C++ template classes is now more reliable when debugging with GDB
    • Android Projects now support NDK development and debugging
    • Unity projects with the .unityproj extension can now be opened directly in SlickEdit
  • Large File Editing
    • Reading and saving files greater than 2GB is now supported. In theory the maximum limit is 2TB, but available system memory and hard drive speed are now the limiting factors. For substantially better performance, use an SSD.
  • General
    • Enhanced special character display: Unicode files now have a separate configuration section to specify how special character sequences are displayed. Multi-character substitutions are supported.
    • Vi/Vim emulation has been re-tooled internally to inherit base functionality from the CUA emulation. The behavior of Vi/Vim-specific key bindings is not affected, but more CUA commands are now available when in Insert Mode
    • Added a new File > Reload with Encoding menu item which enables you to quickly reload an open file with a new encoding.
